# Stack Slide — как собрать оффер, от которого не отказываются

Stack Slide — это кульминация вебинара. На одном слайде видно весь пакет: основной продукт + бонусы + гарантия + цена. Зритель смотрит на него и физически ощущает, что цена — несопоставима с ценностью. Это момент, когда решение «купить или нет» становится очевидным.

## Принцип Grand Slam Offer (Хормози)

Цель: сделать оффер, **от которого невозможно отказаться** — воспринимаемая ценность в 5-10 раз больше цены. Формула:

```
Grand Slam = Core Product × Value Stack × Guarantee × Urgency/Scarcity
             ─────────────────────────────────────────────────────────
                                    Price
```

Каждый элемент усиливает одну из 4 переменных Value Equation:
- **Dream Outcome** — что они получат
- **Perceived Likelihood** — насколько это реально для НИХ
- **Time Delay** — как быстро
- **Effort & Sacrifice** — сколько усилий от них

## Структура Stack Slide

### Часть 1: Core Product (основной продукт)

Флагман из `product-matrix.md`. Показывается как первый пункт стека.

Формат:
```
✓ [Название программы]                         [Value: $X,XXX]
  → [1 строка — что внутри, главное]
```

Пример:
```
✓ SMM-Agency Blueprint (8-недельная программа)   [Value: $3,000]
  → 8 модулей от поиска первого клиента до $10K/мес
```

### Часть 2: Value Stack (бонусы)

3-7 бонусов, каждый с отдельной воспринимаемой стоимостью. Каждый бонус должен **решать конкретную боль** зрителя, которую он высказал бы как «но это же сложно потому что...».

**Типы бонусов по функции:**

| Тип бонуса | Что делает | Переменная Value Equation |
|------------|------------|---------------------------|
| **Ускоритель** | Помогает получить первый результат быстрее | ↓ Time Delay |
| **Шаблон / swipe file** | Избавляет от необходимости создавать с нуля | ↓ Effort |
| **Community / чат** | Дает окружение и ответы на вопросы | ↑ Perceived Likelihood |
| **Дополнительный модуль** | Расширяет результат / глубину | ↑ Dream Outcome |
| **1-на-1 консультация** | Персонализация под его кейс | ↑ Perceived Likelihood |
| **Инструмент / софт** | Автоматизирует часть работы | ↓ Effort |
| **Мини-программа-партнёр** | Дополнительная ниша / навык | ↑ Dream Outcome |

**Формат каждого бонуса:**

```
✓ БОНУС 1: [Название]                              [Value: $XXX]
  → [1-2 строки — что это и для чего]
  → [Почему это важно: какую боль закрывает]
```

Пример:
```
✓ БОНУС 1: First-Client Script Pack                 [Value: $497]
  → 12 готовых скриптов cold outreach + 5 скриптов созвонов
  → Вместо того чтобы учиться писать клиентам с нуля — берёшь шаблон и запускаешь
```

### Часть 3: Guarantee (гарантия)

Гарантия снимает риск. Без неё зритель подсознательно держит «вдруг не сработает». Типы гарантий:

1. **Money-back (базовая):** «30 дней — если не понравится, возврат 100%»
2. **Conditional money-back:** «30 дней — если прошёл всю программу и не получил результат, возврат + $500 за потраченное время»
3. **Result-guarantee:** «Если за 90 дней не выйдешь на первого клиента — работаю с тобой бесплатно, пока не выйдешь»
4. **Better-than-money-back:** «Возврат + можешь оставить материалы» (агрессивная, но сильнейший сигнал уверенности)

Чем конкретнее и «страшнее» для эксперта — тем сильнее. Самый слабый вариант — «гарантия качества» без конкретики (это не гарантия).

**Формат на слайде:**
```
🛡️ 30-ДНЕВНАЯ ГАРАНТИЯ ВОЗВРАТА
Если за 30 дней вы прошли минимум 4 модуля и не получили первого клиента — 
возвращаем 100% оплаты + $500 за потраченное время.
```

### Часть 4: Urgency / Scarcity

**Urgency (дедлайн по времени):**
- «Цена со скидкой только до [дата/время]»
- «Запись на поток закрывается через 72 часа»
- «Сегодняшняя цена действует до конца эфира»

**Scarcity (ограничение по количеству):**
- «Только 50 мест в потоке»
- «Первые 20 человек получают бонус X»
- «После N оплат цена поднимается»

**Правило:** urgency/scarcity должны быть **реальны**. Если «только 50 мест» повторяется каждый запуск — это теряет силу. Если эксперт не может обеспечить реальную scarcity — используем urgency (по времени).

**На слайде:**
```
⏰ Цена действует только до [конец вебинара / полночи сегодня]
🔒 Принимаем только 30 человек в этот поток
```

### Часть 5: Total Value → Anchor → Price

Финальная часть слайда:

```
────────────────────────────────────────
ОБЩАЯ ЦЕННОСТЬ:                 $12,497
────────────────────────────────────────
Моя обычная цена программы:      $5,000
────────────────────────────────────────
⚡ Специальная цена сегодня:      $1,997
     или 3 платежа по $697
────────────────────────────────────────
```

Логика:
- **Общая ценность** ($12,497) — сумма всех элементов стека. Это воспринимаемая ценность.
- **Обычная цена** ($5,000) — anchor. «Я реально продаю это за столько». Делает следующую цену выгодной.
- **Сегодняшняя цена** ($1,997) — конечная. Между якорем и итогом — 2.5x скидка, ощутимая.

**Правило 10x:** Total Value / Price ≥ 10. Если ценность = $12,497 и цена = $1,997 — коэффициент 6.3x. Норм, но можно усилить. Если 10+ — оффер «hellno не могу отказаться».

## Как эксперт рассказывает Stack Slide

Не просто зачитать список. Каждый элемент — мини-история/объяснение:

1. «Первое — это само ядро программы. 8 модулей. Я покажу, что в каждом...»
2. «Но программа без первых клиентов бесполезна. Поэтому БОНУС 1 — это...»
3. «Вы скажете: "а как оформлять сделку, я же не юрист". Закрыто: БОНУС 2...»
4. ... и так далее

Каждый бонус **закрывает одно возражение**, которое зритель даже не успел озвучить. Это создаёт ощущение «он подумал обо всём».

**Время на Stack Slide:** 8-12 минут. Не торопись. Это ключевой слайд.

## Ошибки Stack Slide

- **Слишком много бонусов (10+).** Теряется фокус, каждый бонус обесценивается. Держи 5-7.
- **Бонусы не закрывают конкретную боль.** Бонус «доступ к архиву мастер-классов» — слабый, потому что это больше материала, а не решение боли.
- **Раздутая воспринимаемая ценность.** Если бонус «PDF-чеклист» оценён в $500 — зритель не поверит. Ценности должны быть правдоподобными.
- **Нет гарантии.** Это первое, что выбивает колеблющихся. Гарантия обязательна.
- **Цена названа до стека.** Тогда стек работает как «оправдание цены», а не как создание ощущения «это выгодно».

## Связь со Sales Page

Stack Slide = Sales Page. Если вебинар продал, зритель идёт на страницу оплаты. На этой странице **тот же стек** (тот же порядок, те же формулировки). Иначе — когнитивный диссонанс («на вебинаре обещали одно, а тут выглядит иначе»).

Поэтому Stack Slide — это одновременно и слайд для эфира, и основа для `product-matrix.md` / sales-page.
