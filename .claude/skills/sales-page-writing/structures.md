# Структуры лендингов по архетипам

Готовые скелеты блоков. Берёшь нужный архетип, проходишь по порядку. Опциональные блоки помечены `[опц.]` — включай, если есть фактура и место.

---

## Архетип 1 — Registration (регистрация на событие)

**Задача:** забрать email (+ опц. телефон) за минимальное сопротивление. Трафик уже тёплый — он прошёл прогрев.

**Длина:** 1-2 экрана для одиночного вебинара. 3-5 экранов для challenge/марафона (есть что рассказать про программу дней).

**Разветвление:**
- `webinar` (1-2 экрана) — короткая, классическая
- `challenge` / `marathon` (3-5 экранов) — длиннее, с подробной программой

### 1.1 Webinar (1-2 экрана)

| # | Блок | Обязательно | Что в нём |
|---|---|---|---|
| 1 | **Hero** | ✓ | Hedlайн с обещанием + подзаголовок + CTA + фото эксперта (опц.) |
| 2 | **Программа** | ✓ | 3-5 буллетов «что узнаешь / что получишь» — curiosity bullets |
| 3 | **Когда/где** | ✓ | Дата, время, платформа (Zoom/YouTube Live/закрытый канал), countdown |
| 4 | **Для кого** [опц.] | — | 3-4 пункта «это для тебя, если...» |
| 5 | **Authority** [опц.] | — | 3-5 цифр/фактов про эксперта (результаты, годы практики, кейсы) |
| 6 | **Форма + повтор CTA** | ✓ | Email, имя (опц.), кнопка с тем же текстом, что в hero |
| 7 | **Footer** | ✓ | Политика, оферта, связь |

### 1.2 Challenge / Marathon (3-5 экранов)

| # | Блок | Обязательно | Что в нём |
|---|---|---|---|
| 1 | **Hero** | ✓ | Хедлайн с «за X дней» + подзаголовок + CTA |
| 2 | **Социальное доказательство (цифры)** | ✓ | 3 крупные метрики ($X в продажах / Y учеников / Z лет практики) |
| 3 | **WHAT/WHEN/WHERE** | ✓ | Что это, когда, где, как долго (длительность сессий) |
| 4 | **Программа дней** | ✓ | День 1 — тема. День 2 — тема. … с конкретикой темы каждого дня |
| 5 | **Authority** | ✓ | «Почему слушать меня» — 3-4 пункта с цифрами |
| 6 | **Что получишь** [опц.] | — | Физическая ценность (воркбук, чат, записи) |
| 7 | **Повтор CTA** | ✓ | Большой блок «Save my free spot» |
| 8 | **FAQ** [опц.] | — | 3-4 типовых вопроса (запись? доступ? бесплатно?) |
| 9 | **Форма + финальный CTA** | ✓ | Email + имя |
| 10 | **Footer** | ✓ | Юридика |

### Чек-лист registration

- [ ] Хедлайн содержит **результат + срок** (не только тему)
- [ ] Дата, время и часовой пояс указаны явно (MSK / Local Time)
- [ ] Countdown timer включён (реальный, не декоративный)
- [ ] CTA-кнопка **одинакова** во всех повторах
- [ ] Форма просит максимум email + имя. Телефон — только если реально идёт SMS-напоминалка
- [ ] Нет упоминания цены продукта, который продастся на вебинаре (обещать можно «покажу, как...», но не продавать здесь)
- [ ] Под формой — 1 предложение, что произойдёт (письмо на почту + напоминание за час)

## Архетип 2 — Offer (продающий лендинг)

**Задача:** перевести зрителя/слушателя события в оплату или заявку. Трафик — hot (только что с вебинара / из follow-up) или warm (зашёл по ссылке из прогрева).

### 2.1 Offer → Direct-sale (4-12 экранов)

**Когда использовать:**
- `launch_model = warm` в `launch-strategy.md`
- Цена ≤ 100 000 ₽ (средний чек, не high-ticket)
- Есть готовый платёжный контур (GetCourse / Prodamus / Tilda Payments)
- Продукт понятен без консультации (курс, воркшоп, доступ к сообществу)

| # | Блок | Обязательно | Что в нём |
|---|---|---|---|
| 1 | **Hero** | ✓ | Хедлайн-обещание результата + подзаголовок «для кого» + CTA |
| 2 | **Боль / напоминание** | ✓ | 2-3 абзаца «узнаёшь это?» — возврат к боли из вебинара |
| 3 | **Метод / Big Domino** | ✓ | Ключевая идея вебинара одной фразой + 2-3 предложения обоснования |
| 4 | **Что получишь (Value Stack)** | ✓ | Список модулей/блоков/бонусов с описанием и ценностью. Основа = `product-matrix.md` |
| 5 | **Результат (трансформация)** | ✓ | «Через X недель ты...» — 3-5 пунктов с конкретикой |
| 6 | **Кейсы** | ✓ | 3-5 именных кейсов с цифрами (скриншот результата / отзыв / до-после) |
| 7 | **Гарантия** | ✓ | Текст гарантии из `product-matrix.md`, плашка с иконкой |
| 8 | **Для кого / НЕ для кого** [опц.] | — | 2 колонки «если ты Х — это для тебя / если Y — не подходит» |
| 9 | **Тарифы / цена + anchor** | ✓ | 1-3 тарифа, anchor цена (сумма Value Stack) vs сегодня. Scarcity (если есть) |
| 10 | **FAQ** | ✓ | 6-10 вопросов (из `objections.md`) |
| 11 | **Финальный CTA + deadline** | ✓ | Большая кнопка + таймер / количество мест |
| 12 | **Footer** | ✓ | Юридика, оферта, договор |

**Кнопки оплаты:** если тарифов несколько — каждый со своей кнопкой. Если один — кнопка повторяется после блоков 4, 7, 9, 11.

### 2.2 Offer → Application (4-7 экранов)

**Когда использовать:**
- `launch_model = hybrid` или `cold`
- Цена > 100 000 ₽ (high-ticket)
- Продукт требует квалификации (консалтинг, наставничество, премиум-программа)
- Нужно отсеять неподходящих лидов до созвона

| # | Блок | Обязательно | Что в нём |
|---|---|---|---|
| 1 | **Hero + VSL** | ✓ | Хедлайн с провокацией + подзаголовок «посмотри видео, чтобы подать заявку» + плеер VSL + CTA |
| 2 | **О чём программа (3 столпа)** | ✓ | 3 колонки с иконками: ключевые компоненты ценности (методика / поддержка / инструменты) |
| 3 | **Кейсы (именные, конкретные)** | ✓ | 5-8 карточек: имя + ниша + конкретный результат в цифрах (не «увеличила доход», а «+240К/мес за 8 недель») |
| 4 | **Для кого / НЕ для кого** | ✓ | **Обязательно жёсткое исключение** — «не подавай заявку, если...» (фильтр спам-лидов) |
| 5 | **Как проходит работа (3-4 шага)** | ✓ | 1) подаёшь заявку, 2) созвон-квалификация, 3) решение и старт. Снимает фрикшн «что будет после» |
| 6 | **FAQ** | ✓ | 4-6 закрытых вопросов. **Цена скрыта** — «цена обсуждается индивидуально на созвоне» |
| 7 | **Финальный CTA + мягкая срочность** | ✓ | «Limited spots / следующий набор в ...» |
| 8 | **Footer** | ✓ | Юридика |

**Ключевое отличие от direct-sale:**
- **Цена не указана** нигде на странице. Указание цены = убивает quality application
- **Форма заявки** содержит 5-8 полей: имя, email, телефон, ниша, текущие результаты, что блокирует, бюджет (галочки)
- **CTA** всегда «Apply» / «Записаться на созвон» / «Подать заявку», не «Купить»
- **VSL** почти всегда на hero — без видео конверсия падает кратно

### Чек-лист offer (обе ветки)

- [ ] Конгруэнтность с вебинаром: тот же Big Domino, те же 3 Секрета, тот же язык (пересечение с `content/webinar/<NN>/structure.md`)
- [ ] Value Stack соответствует `product-matrix.md` — не придумываем бонусы, которые не существуют
- [ ] Гарантия — дословно из `product-matrix.md`
- [ ] Минимум 5 именных кейсов с цифрами (не «Мария из Москвы увеличила доход»)
- [ ] Цифры-якоря везде, где можно: сумма Value Stack, anchor-цена, deadline-дата
- [ ] Дисклеймер «результаты не гарантированы» — если применимо (особенно для заработка) в конце после кейсов
- [ ] Для direct-sale: checkout-форма подключена и работает. Для application: заявочная форма в 5-8 полей

## Архетип 3 — Leadmagnet (холодный трафик)

**Задача:** забрать email в обмен на артефакт. Трафик холодный — не знает эксперта, пришёл по рекомендации бота после рилса.

### 3.1 Minimal (Hormozi-стиль, 1 экран)

**Когда использовать:**
- Артефакт self-explanatory (PDF, чек-лист, инструкция)
- Аудитория уже тёплая к нише (нашли через рилс по ключевой боли)
- Бренд эксперта узнаваем (лого + цифра-якорь = достаточно)

| # | Блок | Обязательно | Что в нём |
|---|---|---|---|
| 1 | **Hero** | ✓ | Хедлайн «Get your [артефакт] in under X seconds» + 1 строка подзаголовка |
| 2 | **Визуал артефакта** | ✓ | Мокап PDF / скриншот интерфейса / обложка (правая колонка или под hero) |
| 3 | **Форма + CTA** | ✓ | Email → кнопка «Get my roadmap». Под формой — 1 метрика («Requested by 250 000...») |
| 4 | **Footer (минимум)** | ✓ | Политика конфиденциальности + dislaimer |

**Длина ровно 1 экран.** На мобиле — 1 скролл максимум.

### 3.2 Extended (2-3 экрана)

**Когда использовать:**
- Артефакт требует объяснения ценности
- Soft-ниша (психология, здоровье — нужно больше доверия)
- Бренд пока не узнаваем
- Есть отзывы, которые усиливают (не выдумка)

| # | Блок | Обязательно | Что в нём |
|---|---|---|---|
| 1 | **Hero** | ✓ | Хедлайн-обещание + подзаголовок + CTA |
| 2 | **Визуал артефакта** | ✓ | Мокап + «что внутри» — 5-7 curiosity bullets |
| 3 | **Социальное доказательство** | ✓ | 1-3 отзыва или 1 метрика (количество скачавших) |
| 4 | **Для кого** [опц.] | — | 3-4 пункта «если ты Х — этот PDF для тебя» |
| 5 | **Форма + CTA** | ✓ | Email → кнопка. Под формой — «что будет после» (доставка на email + бот) |
| 6 | **Footer** | ✓ | Юридика |

### Чек-лист leadmagnet

- [ ] Артефакт обещан **конкретно** («PDF на 14 страниц», «чек-лист из 12 пунктов», «видео 28 минут», не «полезные материалы»)
- [ ] Форма просит **только email** (не имя, не телефон — холодный трафик не даст лишнего)
- [ ] Кодовое слово / название воронки из `leadmagnet-funnel.md` — соответствует тому, что зрителям обещали в рилсе
- [ ] Thank-you страница подготовлена (куда падает после submit)
- [ ] Hero работает без прокрутки и без звука — холодный трафик не читает, а сканирует
- [ ] Для extended: не растягиваем ради длины. Если сказать нечего — делаем minimal

## Архетип 4 — Thank-you

**Задача:** подтвердить действие + дать следующий шаг + снять тревогу «что дальше?». Критичная, но игнорируемая страница. Высокий engagement здесь = больше досмотрим вебинара, меньше refund, больше upsell-конверсий.

**Длина:** 0.5-1 экран.

### 4.1 Thank-you → после registration (на вебинар)

| # | Блок | Что в нём |
|---|---|---|
| 1 | **Подтверждение** | «Ты зарегистрирована на [событие] [дата, время MSK]» + конкретика |
| 2 | **Что будет** | «Письмо на [email] через 5 минут. Ссылка на эфир придёт за 1 час до старта» |
| 3 | **Next step (обязательно)** | 1 действие сейчас: добавь в календарь / подпишись на TG / посмотри разогревающее видео (pre-event warmup content) |
| 4 | **[Опц. upsell]** | «Пока ждёшь — вот [мини-курс / PDF / трипваер] за ₽Х» (не для soft-ниш, там этот блок выключаем) |

### 4.2 Thank-you → после application (подача заявки на offer)

| # | Блок | Что в нём |
|---|---|---|
| 1 | **Подтверждение** | «Заявка получена, менеджер свяжется в течение X часов» |
| 2 | **Что будет** | «Позвонит / напишет в WhatsApp на указанный номер» |
| 3 | **Next step** | «Пока ждёшь звонка, посмотри [кейс / разбор / видео]» — укрепление желания |

### 4.3 Thank-you → после оплаты (direct-sale)

| # | Блок | Что в нём |
|---|---|---|
| 1 | **Подтверждение оплаты** | «Оплата прошла. Чек — на email» |
| 2 | **Как получить доступ** | Ссылка на платформу / инструкция / контакт поддержки |
| 3 | **Next step** | «Зайди в чат потока: [link]» / «Смотри welcome-видео» |
| 4 | **Контакт саппорта** | Telegram / email для быстрого ответа |

### 4.4 Thank-you → после leadmagnet

| # | Блок | Что в нём |
|---|---|---|
| 1 | **Подтверждение** | «Письмо с [артефактом] уже на [email]» |
| 2 | **Не пришло?** | Фикс «проверь спам, добавь в контакты» + контакт |
| 3 | **Next step (опц.)** | «Подпишись на канал, где разбираю такие темы глубже» + ссылка на TG / IG |
| 4 | **[Опц. tripwire]** | Мини-оффер за низкую цену (699 ₽ / 1900 ₽) — если воронка это предполагает |

### Чек-лист thank-you

- [ ] Первый экран подтверждает action («Ты зарегистрирована» / «Заявка получена» / «Оплата прошла»)
- [ ] Снята тревога «что дальше?» — явно сказано, что и когда произойдёт
- [ ] Есть **ровно один** next step, не два-три (это размывает engagement)
- [ ] Нет длинного объяснения — страница прочитывается за 10 секунд
- [ ] Есть способ связаться с поддержкой (для paid thank-you — обязательно)

## Общие паттерны, которые работают в любом архетипе

### Hero-формулы (разворачиваются в `headlines.md`)
1. **Результат + срок:** «$373/день за 5 дней»
2. **Провокация + фильтр:** «Обычные люди зарабатывают столько же, сколько врачи»
3. **Скорость + персонализация:** «Get your roadmap in under 30 seconds»

### Плотность CTA
- Registration: 3 повтора (hero / после программы / перед формой)
- Offer direct-sale: 5-7 повторов
- Offer application: 3-5 повторов, все ведут в одну форму
- Leadmagnet minimal: 1-2 (в hero + форма)
- Thank-you: 1 (next step)

### Паттерн «для кого / НЕ для кого»
Два столбца. Слева — «если ты Х / если тебе нужно Y» (5-7 пунктов). Справа — «не подавай заявку, если...» (3-4 пункта). **НЕ для кого** парадоксально увеличивает желание — это техника исключения (подтверждено на Ovens / Hormozi).

### Countdown-таймер
- Registration: обязателен, ведёт к дате события
- Offer: обязателен, ведёт к deadline тарифа / закрытию набора
- Leadmagnet: **не нужен** (артефакт всегда доступен)
- Thank-you: **не нужен**
