# Методология посадочных страниц

Ты пишешь лендинги для воронки запуска блогера. Три главных архетипа и одна служебная страница. У каждого — своя физика, своя длина, свой тип конверсии.

## Четыре типа страниц

| | **Registration** | **Offer** | **Leadmagnet** | **Thank-You** |
|---|---|---|---|---|
| Назначение | регистрация на событие | оплата/заявка на оффер | скачать PDF/видео-разбор | подтверждение + подготовка |
| Стадия воронки | прогрев → событие | событие → оплата | холодный трафик → автоворонка | после любого опта |
| Главный KPI | конверсия в регистрацию (40-65%) | конверсия в оплату/заявку (3-12%) | конверсия в опт-ин (25-55%) | engagement + no-bounce |
| Трафик | тёплый (из прогрева) + ads | тёплый (с вебинара / follow-up) | холодный (рилсы, ads) | уже опт-нуло |
| Температура | warm | hot | cold | — |
| Длина в экранах | 1-5 | 4-12 | 1-3 | 0.5-1 |
| Форма | email (+ phone опц.) | application или прямая оплата | email | нет формы — CTA на действие |
| Ключевой блок | обещание + программа + дата | Value Stack + гарантия + кейсы | обещание + форма | инструкция + next step |

Агент `sales-page-writer` работает в режимах `registration`, `offer`, `leadmagnet`, `thank-you`, `adhoc`. У `offer` — два подрежима: `direct-sale` (цена видна, прямая оплата) и `application` (цена скрыта, заявка → созвон).

---

## Пять правил (общие для всех архетипов)

1. **Один экран = одна задача.** Первый экран = обещание + CTA. Нельзя в hero грузить 5 мыслей. Всё остальное — раскрытие и снятие возражений.
2. **CTA не меняется внутри лендинга.** Одна форма, один текст кнопки, один action. Повторяется 3-7 раз, но всегда про одно.
3. **Конкретика важнее красоты.** «$23.9M в продажах» > «помогаю зарабатывать больше». «Уже 250 000 предпринимателей» > «сотни довольных клиентов».
4. **Возражение снимается до того, как зритель его сформулировал.** Не ждём FAQ — вплетаем ответы в структуру: «для кого / для кого не подходит» до блока цены.
5. **Форма — последний, а не первый порог.** Спрашиваем только то, что реально нужно в следующем шаге. Email + имя для регистрации, всё. Не телефон, не компания, не должность.

## Матрица выбора архетипа

```
Пришла задача → какой лендинг?

Есть событие (вебинар/эфир/марафон/challenge) и нужна регистрация?
  → registration

Есть event-writer output или pitch-секция вебинара, и нужно продавать после события?
  → offer
    ├─ launch_model = warm + price ≤ 100k ₽ + есть чекаут/платёжка → direct-sale
    └─ launch_model = hybrid / cold / price > 100k ₽ / high-ticket → application

Есть leadmagnet-funnel.md, и нужна страница под холодный трафик?
  → leadmagnet
    ├─ Hormozi-minimal (1 экран) — если артефакт сам по себе self-explanatory (PDF, чек-лист)
    └─ Extended (2-3 экрана) — если нужно объяснять ценность, есть отзывы, soft-ниша

После любого опта (регистрация / заявка / оплата) → thank-you
```

**Правило:** если пришла задача писать лендинг, но в state.md нет нужных источников (product-matrix для offer, leadmagnet-funnel для leadmagnet, event из launch-strategy для registration) — верни `blocked: <file> missing`.

## Базовый формат файла лендинга

Каждый лендинг — один `.md` файл в `content/landings/`. Имя файла:

```
<NN>-<type>-<slug>.md
```

Примеры:
- `01-registration-webinar-2026-05-12.md`
- `02-offer-direct-course-flagship.md`
- `03-offer-application-consulting-premium.md`
- `04-leadmagnet-pdf-5-shagov.md`
- `05-thankyou-post-registration.md`

Содержимое файла:

```markdown
---
n: 01
type: registration | offer | leadmagnet | thank-you
subtype: —                         # для offer: direct-sale | application
event_ref: webinar-01              # для registration / offer (связь со event-writer)
product_ref: flagship              # для offer
leadmagnet_ref: pdf-5-shagov       # для leadmagnet
date_published: 2026-05-02
status: my-review
segment: <имя сегмента>
length_screens: 4
---

# [Slug / внутренняя пометка]

## URL / routing

/event/webinar-2026-05-12   # путь в воронке
```

## Чек-лист перед сохранением (общий)

- [ ] Hero-экран читается и решает «остаться/уйти» за 5 секунд
- [ ] Хедлайн конкретный (цифра / срок / результат), не абстрактный
- [ ] CTA повторяется 3+ раз, текст не меняется
- [ ] Форма просит минимум данных
- [ ] Social proof в цифрах, не в словах
- [ ] Возражения закрыты до цены / до формы
- [ ] SEO title + description заполнены
- [ ] Нет длинных тире (только запятые и точки) — как в правилах прочих агентов
- [ ] Нет стоп-слов: «данный», «является», «осуществляется», «уникальный», «лучший в мире»
- [ ] Язык сегмента из `audience-profile.md` (не маркетологский перевод)

**Специфичные чек-листы** для каждого архетипа — в `structures.md`.

## Границы работы

**Скилл покрывает:**
- Тексты и структуру посадочных страниц всех 4 типов
- SEO-мета (title, description, og-image)
- Формулировки CTA и полей формы
- Thank-you страницы

**Скилл НЕ покрывает:**
- Верстку / код / HTML-CSS (это задача дизайнера/разработчика)
- Email-воронку после опта (будущий `email-writer`)
- Сам оффер и Value Stack (это `product-matrix.md` от `product-architect`)
- Сам лид-магнит, серию писем, трипваер (это `leadmagnet-funnel.md` от `launch-strategist`)
- Ads-копирайтинг и baner-тексты (будущий `ads-writer`)

Если пришла задача «сверстай лендинг», «напиши код», «оптимизируй SEO», «сделай email-цепочку» — верни `out_of_scope` с указанием нужного агента или пометкой «не в системе».
