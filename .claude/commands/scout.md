# /scout — структурирование публичных сигналов

Запускает агента `research-scout`. Он не ходит в интернет (это принципиально) — продюсер руками приносит материалы, агент раскладывает их по понятной структуре, из которой потом живут `/audience` и `/reels`.

## Когда звать

- `intake-analyst` пометил в `project-digest.md` секцию «Пробелы» с статусом `critical` по публичным сигналам
- Orchestrator предложил `/scout` перед запуском `/audience`
- Продюсер хочет обогатить ресерч перед касдевом / перед `/reels` в режиме `competitor-analyze`

## Что даёт продюсер

Перед запуском уточни у продюсера:

1. **Что именно принёс** — какие посты / комменты / конкуренты / рилсы
2. **В каком формате** — чистый текст, скриншоты (с OCR-текстом), ссылки, транскрипты голосом
3. **Сколько единиц** — 5 / 20 / 50 материалов

Минимум для полезной работы: 20–30 единиц (комменты считаются по одному).

## Что делает агент

Парсит переданное и раскладывает по 4 файлам в `brief/public-signals/`:

- **`public-comments.md`** — комментарии подписчиков под постами эксперта. Самое ценное — это **реальный язык аудитории**: каким словами они говорят о своей проблеме
- **`top-posts.md`** — посты эксперта, которые залетели (охваты / реакции / комменты). Показывают, что резонирует
- **`reels-transcripts.md`** — расшифровки чужих / своих рилсов. Используется `/reels` в режиме `competitor-analyze`
- **`competitors-content.md`** — контент прямых конкурентов: что продают, как позиционируются, какие оффера

Каждая запись структурирована: источник / дата (если известна) / содержимое / что извлекается (инсайт или цитата).

## Precheck

- Продюсер реально принёс материалы (нельзя запустить «пусто — авось что-то найдёт»)
- Материалы не являются личной / приватной перепиской — только публичное

## Выход

- Обновлённые файлы в `brief/public-signals/`
- Короткий отчёт: сколько единиц принято, что самое ценное, что дальше делать

## После завершения

1. Перезапустить `/intake` — чтобы digest пересобрался с новыми сигналами
2. После этого → `/audience` (Проход 1)

**Важно:** `/audience` читает digest, а digest читает public-signals только через intake-analyst. Поэтому между `/scout` и `/audience` обязательно должен стоять повторный `/intake`.

## Типичные ошибки

- Запустить без материалов или с 2–3 штуками — агент не сможет выявить паттерны, и работа пустая
- Приносить скриншоты без текста (если OCR не настроен) — агент их не прочитает
- Забыть перезапустить `/intake` после — следующие агенты не увидят новые сигналы
- Использовать как замену касдеву — `/scout` показывает, что люди говорят публично. Касдев показывает, что они думают на самом деле. Это разные уровни глубины
