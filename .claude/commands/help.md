Покажи продюсеру шпаргалку по системе. Формат:

```
Мультиагентная система запуска — шпаргалка

💼 Продажа (до подписания)
  /presale             Pre-sale стратегия для потенциального клиента (одна страница)

📁 Старт
  /new-project         Завести нового блогера (опрос + скелет проекта)

📊 Диспетчер
  /status              Что сейчас происходит, куда дальше
  /next                Выполнить следующий шаг по плану
  /review              Перевести артефакт в следующий статус

🔍 Onboarding / Discovery
  /intake              Парсинг brief/ → project-digest.md
  /scout               Структурировать принесённые публичные сигналы
  /audience            Сегментация ЦА + money-сегмент
  /castdev             Запустить 3-4 дневный касдев-прогрев (TG + IG)

🎯 Strategy
  /product             Спроектировать флагманский оффер (Grand Slam)
  /strategy            Стратегия запуска + таймлайн + юнит-экономика

✍️ Preparation (контент)
  /event               Вебинар / эфир / ВСЛ
  /tg                  Прогрев в Telegram (посты)
  /ig                  Прогрев в Instagram (сторис, лента, карусели)
  /reels               Сценарии Reels / TikTok
  /landing             Посадочные страницы воронки

ℹ️ Справка
  /help                Эта шпаргалка
```

После шпаргалки коротко напомни:
- Фазы идут последовательно: Onboarding → Discovery → Strategy → Preparation → Execution
- Все продающие артефакты (audience-profile, product-matrix, launch-strategy, offer-лендинг, вебинар) обязательно идут на client-review эксперту перед запуском
- Подробный гайд — в `README.md` в корне репо
