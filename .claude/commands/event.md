# /event — сценарий события

Запускает агента `event-writer`. Это самый важный контент запуска — **событие продаёт**, остальное лишь прогревает к нему. Большинство продаж в warm-модели приходит в 10-дневное окно после события.

## Когда звать

- `research/event-brief.md` готов (launch-strategist Проход 2 завершён)
- Все ключевые артефакты утверждены: `audience-profile.md`, `product-matrix.md`, `launch-strategy.md`
- `data/warmup-plan.md` готов — event-writer знает, с каким состоянием аудитории работает

## Методология

Синтез четырёх школ под один сценарий:

- **Brunson (Perfect Webinar)** — каркас: Epiphany Bridge, 3 Secrets, Stack Slide, Close
- **Fladlien (One-to-Many)** — интрига на весь эфир, микро-закрытия, сдвиг убеждений аудитории
- **Hormozi ($100M Offers)** — Value Equation и Stack в pitch-секции
- **Gadzhi (VSL-style)** — визуальный ритм, B-roll, дикция

Ключевая формула: **контент = продажа**. Никакого разделения «сначала бесплатная польза, потом оффер» — обучающая часть подводит к офферу изнутри, не после.

## Режимы

Можно запустить по шагам или сразу всё:

- **`structure`** — скелет вебинара с таймингами (всегда первый шаг)
- **`script`** — полноценный сценарий со speaker notes для каждого блока
- **`slides`** — слайды: визуал + текст на слайде + speaker notes
- **`pitch-only`** — только продающая секция (оффер + Stack + close + Q&A)
- **`adhoc`** — точечная правка существующего вебинара

**По умолчанию:** `structure` → продюсер согласовывает → `script` → `slides` → `pitch-only`. Пошагово — чтобы можно было корректировать направление без переписи всего.

## Precheck

Агент не стартует, если:
- `research/event-brief.md` отсутствует — нет ТЗ (формат, длительность, главное обещание)
- `audience-profile.md` или `product-matrix.md` не в `done` — сценарий будет бить мимо

## Вход

- `research/event-brief.md` — ТЗ (формат, обещание, 3 секрета, Epiphany Bridge, кейсы, оффер)
- `data/audience-profile.md` — язык сегмента, боли, возражения
- `data/product-matrix.md` — оффер, цена, гарантия, бонусы
- `data/warmup-plan.md` — что уже рассказали в прогреве (не дублировать)
- `research/project-digest.md` — тон эксперта, origin story

## Выход

- `content/webinar/<NN>-<slug>/structure.md` — каркас
- `content/webinar/<NN>-<slug>/script.md` — сценарий
- `content/webinar/<NN>-<slug>/slides.md` — слайды
- `content/webinar/<NN>-<slug>/offer-pitch.md` — питч

**Обязательно** → `client-review` для `script.md` и `offer-pitch.md`. Эксперт должен прочитать сценарий вслух и подтвердить, что это его голос.

## После завершения

1. Прогнать сценарий с экспертом голосом (репетиция 1–2 раза)
2. Собрать / отрисовать слайды
3. Подтвердить техническую площадку (Zoom / YT / TG Live)
4. Запустить регистрацию через `/landing` (режим `registration`)

## Типичные ошибки

- Писать скрипт без event-brief — теряется связка со стратегией, оффер висит в воздухе
- Разделять контент и продажу — «сначала польза, потом продам» = слабый pitch и провал в окне
- Не репетировать вслух — эксперт на эфире сбивается с чужих формулировок, теряет уверенность
- Перегружать слайды текстом — они опора, а не суфлёр. Максимум 15–25 слов на слайд
- Забить на offer-pitch — это блок, где деньги, его нельзя писать «на глазок»
