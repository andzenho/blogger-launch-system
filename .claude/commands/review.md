# /review — смена статуса артефакта

Управление жизненным циклом артефакта. В системе 4 значимых статуса — команда переводит между ними, обновляет `state.md`, записывает в журнал событий.

## Жизненный цикл артефакта

```
draft  →  my-review  →  client-review  →  done
         (продюсер)    (эксперт)       (согласован)
```

- **`draft`** — агент только что завершил работу, артефакт сырой. Продюсер ещё не смотрел
- **`my-review`** — продюсер читает сам, ищет очевидные ошибки / неточности / нестыковки с брифом
- **`client-review`** — отдан эксперту. Ждём подтверждения или правок
- **`done`** — эксперт утвердил. Можно опираться в следующих фазах
- **`правки`** — особая ветвь: из `my-review` продюсер отправляет обратно автор-агенту со списком правок

## Когда звать

- Прочитал черновик сам, всё ок → `my-review → client-review`
- Эксперт ответил «ок / согласен» → `client-review → done`
- Эксперт вернул с правками → `client-review → правки` (перезапуск автора с правками)
- Нашёл ошибки сам → `my-review → правки`

## Что делает команда

1. **Спрашивает:** какой артефакт и какой переход. Возможные артефакты:
   - `audience-profile.md` (обязательный client-review)
   - `product-matrix.md` (обязательный client-review)
   - `launch-strategy.md` (обязательный client-review)
   - Сценарий вебинара в `content/webinar/<NN>-<slug>/` (script + offer-pitch — обязательный client-review)
   - Offer-лендинг в `content/landings/` (обязательный client-review)
   - Прочие артефакты (прогревы TG/IG/Reels, registration-лендинг) — client-review по желанию
2. **Обновляет поле `status`** внутри самого артефакта (если есть шапка)
3. **Обновляет `state.md`** через orchestrator — ставит новый статус фазы + добавляет запись в журнал событий
4. **Подсвечивает** разблокированные шаги: если `audience-profile` ушёл в `done`, команда напомнит, что теперь можно звать `/product`. Но **не запускает автоматически** — продюсер решает, когда начать

## Что обязательно идёт на client-review

Без «ок» эксперта **нельзя двигаться дальше**:

- `audience-profile.md` — блокирует `/product`
- `product-matrix.md` — блокирует `/strategy`
- `launch-strategy.md` — блокирует `/event` и старт фазы Preparation
- Сценарий события (script + offer-pitch) — блокирует проведение эфира
- Offer-лендинг — блокирует запуск окна продаж

## Типичные ошибки

- Ставить `done` на client-review без явного согласия эксперта («он не ответил, значит согласен») — потом приходится переделывать на ходу
- Не дожидаться `done` по `audience-profile` и запускать `/product` — оффер уйдёт в сегмент, который эксперт не подтвердил
- Забыть обновить статус внутри самого файла (только в `state.md`) — в следующей сессии orchestrator увидит рассинхрон и зависнет
- Не писать в журнал событий, что именно эксперт правил — теряется контекст, следующие правки противоречат предыдущим
