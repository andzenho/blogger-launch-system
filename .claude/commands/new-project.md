# /new-project — старт нового блогера

Единственная команда, которая работает из корня репозитория. Заводит изолированную папку под нового клиента со всеми шаблонами. После неё вся дальнейшая работа ведётся из `projects/blogger-[имя]/`.

## Когда звать

- Продюсер решил взять нового блогера в работу (или pre-sale закрыт, `/presale` привёл к сделке)
- Ещё не создана папка `projects/blogger-[имя]/`

## Что происходит

### Шаг 1. Опрос — 8 вопросов, по одному

Команда задаёт их по очереди, ждёт ответа на каждом:

1. **Имя блогера** (латиницей — для названия папки)
2. **Короткое описание ниши** (1–2 строки)
3. **Тип ниши** — твёрдая (измеримый результат: доход, профессия, навык) или мягкая (состояние, отношения, духовность)
4. **Есть ли база** (TG-канал, Instagram, email) — размер каждой площадки
5. **Ссылки на соцсети** (TG, IG, YouTube, Threads, TenChat и т.д.)
6. **Формат сотрудничества** — % с продаж / фикс / гибрид + грубые условия
7. **Целевой период запуска** — конкретный deadline или «не горит»
8. **Особые правила / ограничения** — что нельзя говорить, нежелательные форматы, табу-темы, прошлые конфликты

### Шаг 2. Скелет проекта

После всех ответов:

- Копирует `_TEMPLATE/` → `projects/blogger-[имя]/`
- Заполняет `projects/blogger-[имя]/CLAUDE.md` реальными данными — все `[TBD]` заменяются на ответы продюсера
- Заполняет блок `config` в `projects/blogger-[имя]/state.md`:
  - `niche_type`: hard / soft
  - `has_base`: yes / no (определяется по ответу на вопрос 4)
  - `launch_model`: оставляет `TBD` — определится в фазе Strategy
  - `castdev_form_url`: оставляет пустым — продюсер впишет после создания формы на шаге Discovery

### Шаг 3. Доклад продюсеру

- **Папка создана:** `projects/blogger-[имя]/`
- **Следующий шаг:** клиент заполняет `brief/client-intake.md` (версию hard или soft по `niche_type`) и даёт транскрипт разговора в `brief/discovery-call.md`
- **Далее:** продюсер перезапускается из папки проекта (`cd projects/blogger-[имя]/`), запускает Claude оттуда, и вызывает `/intake` как только материалы брифа будут на месте

## После завершения

1. Перейти в папку проекта: `cd projects/blogger-[имя]/`
2. Отдать эксперту `brief/client-intake.md` — анкету (hard или soft версия под `niche_type`)
3. Записать с экспертом discovery-call, положить транскрипт в `brief/discovery-call.md`
4. Запустить `/intake`

## Типичные ошибки

- Запускать `/new-project` из папки уже существующего проекта — создаётся вложенная папка и всё путается
- Ввести имя кириллицей — ломает пути, на macOS могут быть проблемы с кешами
- Не заполнить `CLAUDE.md` сразу — потом забывается, и агенты работают на обобщённых настройках
- Запустить `/intake` до того, как эксперт заполнил анкету и прислал транскрипт — digest соберётся пустой, с большими пробелами
