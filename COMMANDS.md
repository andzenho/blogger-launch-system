# Шпаргалка по командам

Короткий гайд по всем командам системы. Держи под рукой, пока не запомнишь.

> Все команды запускаются из папки проекта (`projects/blogger-[name]/`).
> Исключение — `/new-project`, её можно звать из корня.

---

## Как устроено

Команда = короткий алиас. Ты пишешь `/audience` — внутри запускается нужный агент с нужной методологией. Тебе не надо помнить, как его зовут и какие у него входы.

Фазы идут по порядку:

```
Onboarding  →  Discovery  →  Strategy  →  Preparation  →  Execution
```

Перепрыгивать нельзя. Каждая следующая фаза опирается на артефакты предыдущей.

---

## До подписания клиента

### `/presale`
Собирает pre-sale стратегию — короткий план запуска (1–2 экрана) для потенциального эксперта, с которым ещё не подписан договор. Используется, чтобы закрыть сделку.

**Что происходит:**
1. Система задаёт минимум вопросов о клиенте (ниша, база, цель, условия)
2. Пишет документ: окна продаж → декомпозиция → тарифы → условия → таймлайн неделями → перспектива
3. Сохраняет в `projects/_presale/blogger-[имя]-pitch.md`

**Когда звать:** у тебя есть созвон с потенциальным экспертом, нужно показать «вот как мы могли бы работать».

**Важно:** это не запуск проекта. Настоящую работу стартуем через `/new-project` только после согласия эксперта.

---

## Старт нового блогера

### `/new-project`
Заводит нового клиента с нуля.

**Что происходит:**
1. Система задаёт 8 вопросов (имя, ниша, база, соцсети, условия, дедлайн, ограничения)
2. Копирует скелет из `_TEMPLATE/` в `projects/blogger-[name]/`
3. Заполняет `CLAUDE.md` и `state.md` твоими ответами
4. Говорит, что отдать клиенту дальше

**После этого:** отдаёшь эксперту `brief/client-intake.md` (твёрдая или мягкая версия) и просишь записать `brief/discovery-call.md`.

## Диспетчер

Три команды, которые помогают не потеряться.

### `/status`
Покажет, на какой фазе мы сейчас, какие артефакты готовы, что в ревью, что следующее. Ничего не делает — просто рассказывает.

**Зови, когда:** вернулся к проекту после паузы и забыл, где остановились.

### `/next`
Делает следующий шаг сам. Система смотрит в `state.md`, решает, какого агента звать, и запускает.

**Зови, когда:** хочешь двигаться по плану, не думая о том, кого вызывать.

### `/review`
Переводит артефакт по статусам:

```
draft  →  my-review  →  client-review  →  done
```

**Зови, когда:**
- Проверил черновик сам → `my-review → client-review`
- Эксперт дал «ок» → `client-review → done`

## Onboarding и Discovery

Задача фазы — понять блогера и его аудиторию до деталей.

### `/intake`
Парсит всю папку `brief/` (анкета клиента, транскрипт созвона, материалы) и собирает `research/project-digest.md` — один файл-правду для всех остальных агентов.

**Когда звать:** после того, как клиент заполнил анкету и прислал запись созвона.

### `/scout`
Структурирует публичные сигналы, которые ты принёс руками — посты блогера, комменты подписчиков, рилсы, контент конкурентов. Кладёт в `brief/public-signals/`.

**Когда звать:** если `/intake` пометил, что публичных данных не хватает.

### `/audience`
Сегментация аудитории. На выходе — `data/audience-profile.md`: сегменты, денежный сегмент по Хормози, Dream Outcome и Problems List на каждый сегмент.

Работает в два прохода:
- **До касдева** — гипотезы + ТЗ на касдев-посты
- **После касдева** — финальный профиль с фактами от живых людей

### `/castdev`
Запускает касдев-прогрев на 3–4 дня: параллельно в TG и IG, с анкетой через Google-форму и CTA на разбор/консультацию.

**Правила касдева:**
- Анкета — через форму, не в комментах
- 3–4 дня прогрева, TG + IG параллельно
- Бонус за заполнение — выбирает продюсер
- Финальный вопрос — на разбор/консультацию
- Цену не называем
- CTA ведёт в форму

## Strategy

Фаза, где мы собираем оффер и план запуска. Оба артефакта обязательно идут на `client-review`.

### `/product`
Проектирует один флагманский оффер по методологии Grand Slam Хормози: Value Equation, Value Stack, бонусы, гарантии, scarcity, urgency, название по MAGIC. Цена — из скора Purchasing Power в audience-profile.

**Вход:** утверждённый `audience-profile.md`
**Выход:** `data/product-matrix.md`

### `/strategy`
Проектирует запуск: таймлайн, концепция прогрева по 4 волнам, трафик-план, 2–3 варианта формата события, юнит-экономика, KPI.

**Вход:** audience-profile + product-matrix + `launch_model` из state.md (cold / warm / hybrid)
**Выход:** `data/launch-strategy.md`

## Preparation — контент запуска

Когда стратегия согласована, пишем сам контент.

### `/event`
Вебинар / эфир / ВСЛ. Синтез четырёх школ: Brunson (Perfect Webinar), Fladlien (One-to-Many), Hormozi ($100M Offers), Gadzhi (VSL-стиль).

На выходе — структура, speaker notes, слайды, pitch-секция (оффер + Stack + close), Q&A.

### `/tg`
Прогрев в Telegram-канале эксперта. Режимы:
- `warming-plan` — индекс постов по 4 волнам
- `warming-write` — сами тексты
- `castdev` — прогрев под касдев
- `adhoc` — отдельный пост под задачу

### `/ig`
Прогрев в Instagram. Пишет сторис-цепочки (1 день = 1 сюжет по технике ЗАМЕТКА), подписи к постам ленты, тексты каруселей, промо под эфиры. Места под рилсы планирует, но сценарии — в `/reels`.

### `/reels`
Сценарии Instagram Reels и TikTok. Две цели:
- **Прогрев** — привязан к волнам W1–W4 из warmup-plan
- **Трафик на лид-магнит** — холодная аудитория, кодовое слово → воронка

Режимы: `write`, `adhoc`, `leadmagnet-batch`, `competitor-analyze`.

### `/landing`
Посадочные страницы под воронку запуска. Четыре типа:
- `registration` — лендинг под регистрацию на вебинар/эфир/марафон
- `offer` — продающий лендинг после события (подрежимы: `direct-sale` и `application`)
- `leadmagnet` — холодный трафик на PDF/чек-лист
- `thank-you` — подтверждение после опта

## Типичный цикл работы

```
1.  /new-project           → завели блогера
2.  Клиент заполняет brief → client-intake + discovery-call
3.  /intake                → собрали digest
4.  /audience (1-й проход) → гипотезы + ТЗ на касдев
5.  /castdev               → 3–4 дня прогрева в TG+IG
6.  /audience (2-й проход) → финальный профиль
7.  /product  →  /review   → клиент смотрит оффер
8.  /strategy →  /review   → клиент смотрит стратегию
9.  Параллельно:  /event /tg /ig /reels /landing
10. Execution              → трафик и событие по таймлайну
```

В любой момент:
- `/status` — где мы сейчас
- `/next` — сделай следующий шаг
- `/help` — напомни команды

---

## Что обязательно уходит на client-review

Продающие артефакты без «ок» от эксперта не идут в работу:

- `audience-profile.md`
- `product-matrix.md`
- `launch-strategy.md`
- Offer-лендинг
- Сценарий вебинара

Всё остальное (прогрев, рилсы, посты) — показываем по ходу.
