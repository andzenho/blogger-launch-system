# Multi-agent Launch System — корень

Это корень системы запусков с блогерами. Здесь живут:
- `.claude/agents/` — агенты (intake-analyst, audience-analyst, product-architect, launch-strategist, event-writer, telegram-warmer, instagram-warmer, reels-writer, sales-page-writer, research-scout, orchestrator)
- `.claude/skills/` — методология (telegram-warming, instagram-warming, viral-reels, webinar-writing, sales-page-writing, launch-strategy)
- `_TEMPLATE/` — скелет нового проекта
- `projects/blogger-[name]/` — изолированные папки клиентов

**Данных конкретных клиентов в корне быть не должно.** Мозги агентов — в `.claude/agents/`, методология — в `.claude/skills/`, общие уроки — в `.claude/lessons/` (если появится).

## Где что искать

Агенты в файлах ссылаются на методологию как `.claude/skills/...` — это путь от корня репо. Работа обычно ведётся из `projects/blogger-[name]/`, откуда `.claude/` находится walk-up'ом (поднимаясь на 2 уровня). Если агент не нашёл файл — он должен подняться к корню (через абсолютный путь или `../../.claude/...`).

## Правило работы

Работа над конкретным блогером ведётся из папки проекта: `cd projects/blogger-[name]/`, затем запуск Claude. Проектный `CLAUDE.md` перекрывает этот общий.

Если сессия открыта из корня — не читать содержимое `projects/*/`. Только помогать с системой, скелетом, агентами.

## Архитектура

Всё важное описано выше: агенты — в `.claude/agents/`, методология — в `.claude/skills/`, скелет — в `_TEMPLATE/`, проекты — в `projects/blogger-[name]/`. Отдельного `architecture.md` не держим — если появятся общие уроки между проектами, складывай их в `.claude/lessons/` отдельными файлами по темам.

Визуальная карта системы (фазы, поток артефактов, skills→agents) — в [.claude/system-map.md](.claude/system-map.md). Это справочник для человека, автоматически в контекст не грузится.

## Старт нового проекта

Когда пользователь говорит «новый проект» / «заводим нового блогера» / аналог — задай эти 8 вопросов по порядку:

1. Имя блогера (латиницей, для названия папки)
2. Короткое описание ниши (1–2 строки)
3. Тип ниши — твёрдая или мягкая
4. Есть ли база (ТГ-канал, Instagram, email)? Приблизительный размер
5. Ссылки на соцсети (TG, IG, YouTube и т.д.)
6. Формат сотрудничества (% с продаж / фикс / гибрид) + грубые условия
7. Целевой период запуска (deadline или «не горит»)
8. Особые правила / ограничения (что нельзя говорить, нежелательные форматы)

После ответов:
1. Скопировать `_TEMPLATE/` → `projects/blogger-[name]/`
2. Заполнить `projects/blogger-[name]/CLAUDE.md` данными опроса
3. Заполнить `config` блок в `projects/blogger-[name]/state.md`
4. Сказать клиенту: отдай `brief/client-intake.md` (твёрдая или мягкая версия) и транскрипт `brief/discovery-call.md`
5. Дальше работать уже из папки проекта
